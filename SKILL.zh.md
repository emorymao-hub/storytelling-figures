# storytelling-figures

把**一段对话里聊出来（或正卡着想搞懂）的概念** → 一个 `figure-prompt.md`（纯文本 prompt），粘给 GPT image-2 / nano-banana 出**叙事/概念/科普/机制图**。产物永远是 `.md`，Claude 和 Codex 都能用。

## 运行契约 · 先读这个（每次照走；与下文任何冲突，以本节为准）

**四步走：**

1. **脑暴 + 把内容弄对** —— 收割对话 / 做答案，把知识点全摊开、确保机制不错（**先对，后画**）。
2. **三个选择（🛑 三道硬闸，选完才出图）** —— ① 知识点(≤5) ② 套餐(科普/PPT/详细/journal) ③ 强调(侧重比较/秒懂/讲解/机理……)。
3. **整合成 prompt** —— 把必要信息装配成 prompt，**别限太死、给 image-2 发挥空间**。
4. **safeguard** —— 出图前过一遍红线自检。

和用户的**交互只有第 2 步那三下**；第 1 步收割、第 3 步整合、第 4 步自检都是你做的。

**🛑 三道闸每次必弹、用户选完才出图。** 唯一例外 = 用户明说"你定 / 直接画 / 别问了"。**别因为"内容很清楚"就自己拍板、闷头出图**（最常见的翻车）。

**构图交给模型。** 第 2 步的 ③ 只给一句"强调什么"的 note，**不画 ASCII、不定面板数/左右上下**——版式让 image-2 结合这道题自己发挥（实测：钦定布局反而比自由体更差）。

**两条硬约束（正确性，永远开）：** ① 所有文字一次画全（含中文、不留空、接受乱码）；② 防 prompt 指令外泄进画面。

**交付看平台：** Claude 只交 `figure-prompt.md`、不出图；Codex 交 prompt ＋ 原生文生图（在三道闸答完之后）。**永不手搓 SVG/PNG 充数。**

**只画**机制/概念/流程示意图；数据图表 → `nature-figure`，整套 PPT → `academic-pptx`。

> **⚠️ 最高律（覆盖本文件一切其它规则）：每张图都不一样、各讲各的故事，且发挥 image-2 画得好看。千图一面 / 简陋方框 = 彻底失败。** 约束只有两层：**硬约束**（上面那两条，正确性，永远开）+ **灵感库**（配色/版式/质感/隐喻……菜单不是清单，现场挑几样、变着用，每张图给独有 signature）。

## 身份（被叫起来先记住）

- **你多半被一段对话半路叫起来**：用户聊着才想画，或聊透了要画给人看。**第一件事是接住对话、收割已有内容，不是从头发问卷。**
- **调用透明**：第一句工作消息就明说"我会用 storytelling-figures 把刚才内容整理成出图 prompt/图"（按平台措辞微调），别静默画图——用户必须能判断这次到底调没调用 skill。
- **一律可点选项 + 留"你定"兜底**：所有提问都用**可点选项**（把你的推断放第一个）+ 永远留一项「**按最佳方式呈现（你定）**」，**绝不自由问答逼用户打字**。允许用户看不懂/不想选，由你挑最优直接做。
- **读语气定问多少**：从用户的语气/用词推断他的知识程度和诉求清不清楚——**内行/外行/诉求很具体 → 少问甚至不问；含糊看不出 → 一律当不清楚，要问**（解释类问 1–3 个、讲解/产出类多问几个像启发式引导）。⚠️ **少问 ≠ 跳过三道硬闸**：闸是用户的呈现选择，"少问"只是把推断当第一项让他一点即过，不是替他拍板。
- **上下文优先，别重问已知**：先看对话已定了什么（平台、是否在做 PPT、用途、能否推断 focus）。已知的不问；能推断的用"把推断当第一个选项"快确认，别从零发问。
- **语言跟用户的请求走**：问中文答中文、问英文答英文，**图里标签也用那个语言**——别因 skill 是中文写的、或项目默认中文，就一律中文回。用户另有要求（如"图用英文"）按用户的。
- **跨平台（Claude / Codex 通用）**：本文只说"动作"不绑工具——"用**可点选项**问"（Claude 用 `AskUserQuestion`，Codex/其它纯文本列选项）、"**派研究子 agent / 查官方源**核实"（Claude 用 `claude-code-guide`，Codex 用自带研究/检索）。

> 边界：按平台交付，但永远**不手搓 SVG/PNG/HTML 充数**；不做数据图表（→ `nature-figure`）；不做整套 PPT（→ `academic-pptx`）；不保"精准换单个元件"。

## 第 1 步 —— 脑暴 + 把内容弄对（先对，后画）

**目标：把这张图要讲的东西全摊开，且机制确保正确。** 没把内容弄对之前，不许进第 2 步。

1. **收割对话 → 内容地图 + 白名单**：把已聊出的内容/机制整理成「内容地图」，同时建「**可见内容白名单**」（`允许画/写的词与实体` / `禁止混入的对话泛称` / `不确定需问·需核实的词`）。例：用户只要求解释 `fucntion → function`，白名单就只有 `fucntion`、`function`、相关机制词；对话里泛指 AI 的"模型"不准变成"模型方向"或 `logit(模型)`。
2. **内容先行（content-first）**：图是**已经做出来/已聊清楚的答案**的可视化，不是话题装饰。每个将来要画的元件都对应一条真实内容。
3. **深档要钻到实现层、可溯源**：研究/最深档、或理解型要钻实现层时，Phase A 必须钻到**具体数据结构 / 算法 / 判定分支**，每条**可溯源**。**禁用"识别 / 匹配 / 相似度 / 处理"这类含糊动词糊过去**（真翻过车：把"默认=注意力直选、无检索"误写成"相似度打分"）；**不许"我恰好知道"**——真去查官方源 / 派一个**研究子 agent** 核实，把真机制名（attention、余弦相似度、top-k、动态注入…）落到图上（机制/数值必须可溯源）。
4. **冷启动兜底**：少数情况上来就"帮我画个 X"、没有前置对话可收割——这时才自己从零做答案（Phase A：把概念真机制过准过全、产内容地图），再走第 2 步。

---

## 第 2 步 —— 三个选择（🛑 三道硬闸，选完才出图）

> 🛑 **「内容清楚」≠ 可以跳过。** 这三道定的是**呈现**（突出哪几块、哪个套餐、强调什么）——**即使内容 100% 清楚，仍是用户的选择**。**绝不许自己判断"很明确了"就替用户定、闷头出图**（实测翻车：Codex 自作主张按"详细教学图"直接生成、没给任何选项）。**唯一例外 = 用户明说"你定 / 直接画 / 别问了"。** 注意：第 3 步把**构图**交给模型是另一回事——**呈现选择归用户，构图归模型，都不归你（agent）替用户拍板。**

每道都用**可点选项**（把你的推断放第一个）+ 留「你定」兜底；有 `AskUserQuestion` 用卡片，没有就纯文本编号列（`1. … 2. … 6. 你定`）。

### ① 知识点（≤5）

被调用后**立即脑暴**：把内容里**所有可能的知识点/子主题**摊出来，问"这张图想突出哪几块？"。用户勾选 → 锁定"图要装什么"。
- **绝不许跳过、绝不许替用户默认 focus**——漏这步直奔形式 = 最常见的翻车。
- 选项**从对话内容现场枚举**（不写死）；可多选，但提示**一张图最好 ≤3–4 个知识点**，多了糊。

### ② 套餐（一步定"画多满 + 长什么样"）

把散开的 depth / style / coverage 收成一个"套餐"选择——用户选**目的地**，各轴默认就捆好了。留「其他/自行描述」逃生口。

| 套餐 | 给谁/干嘛 | 留白 | 最少元件 | 内部细节 | 一句目标 |
|---|---|---|---|---|---|
| **科普** | 外行 / 海报 | 45–60%（透气） | 1 主对象 + 2–4 标签 + 1 主线 + 1 句 caption | 主对象画得形象，其余从简 | 3 秒懂一个概念 |
| **PPT** | 报告幻灯 | 25–40% | 3–6 个主部件，各带小标题 + 1–2 行说明 + 清晰主流向 | 每部件画成真实对象、有内部结构，不堆顶 | 边讲边指，听众跟得上 |
| **详细描述** | 教学/自学的密集讲解图 | **≤20%，空白=没画完** | **≥6–10 个有内容区块**，每区有子结构/子步骤 | **每个概念露出内部机制**（如 attention 画到 QKV/softmax 条/残差加法），禁空盒 | 不靠口头也自洽读懂 |
| **journal-figure 启发** | 投稿论文 figure | ≤20%，克制 | 分面板 a/b/c + 图例 + caption，出版规范齐全 | 每面板用真实图元、露机制，克制配色(如 Okabe-Ito) | 审稿人级 |

**三条硬规则（专治"详细不够详细"）**：① 详细/journal 档**留白 ≤20%，把留白当缺陷信号**——看到空白就继续填；② 详细/journal 档**每个区域必须露内部结构**，禁空盒子/纯标签方块；③ 每档都按"最少元件数"下限给（可数 floor），堵偷懒。
> 但 **清晰 > 完整(floor) > 密度**：详细档再满也得**严格易读**，不许糊；不为"像 Figure 1"硬加密。
> 区分 **详细 vs journal**：详细=密度顶满的"讲解图"（画风自由、可上色可隐喻）；journal=出版规范的"论文图"（克制、分面板、克制配色）。

**journal 套餐的「三保」**（正式场景最易翻车）：① **保求异**——靠"真实图元 + 语义配色(一模态一色) + signature=真实焦点"差异化，prompt 明令避开 image-2 俗相（渐变方框流/发光节点/假 3D 铬光）；② **保正确**——元件来自内容地图或用户原话，可疑处核实，**绝不为构图编出方法里没有的步骤/箭头**；③ **保不胡写**——标签逐字来自真实术语、绝不臆造，拿不准宁可问/留白（正式投稿最终文字保真靠矢量工具，image-2 只交布局灵感）。

### ③ 强调点（**取代旧"选布局"闸**）

只问一句**"这张图侧重什么"**，**构图交给模型**结合这道题发挥——不再让用户挑版式、不画 ASCII。
- 给 **2–4 个"侧重 X"方向（启发式，按这道题的内容现场发挥、别写死）**，典型如：**侧重比较**（并列例子谁不同）/ **侧重秒懂**（一眼抓住核心）/ **侧重讲解**（一步步走清流程）/ **侧重机理**（钻进内部机制）……留「你定」兜底。
- 用户选的强调 → 写成一句 **note 注入 prompt**（"重点体现 X、弱化 Y"），**只定轻重、不定摆放**：archetype、嵌套、面板数、左右上下**全交给 image-2**。
- **不画 ASCII、不钦定版式**——实测钦定布局会掐死 image-2、比自由体还差（这正是把旧"选布局方向"闸换成"选强调"的原因）。
- ⚠️ **note 是给模型的构图指引，不写进图里当文字**（除非用户要它当 caption）。

### Deck 模式（旁路：一套图 / 多页 / 系列图，不是单张）

任务是**一套图**而非单张时，别把单图流程 ×N（会变成几十个问题）。改成"**默认设一次 + 每页只问会变的**"：
- **deck 级默认，开场设一次、之后不重问**：意图、style 家族 + 密度、house style（统一视觉线，保证 N 页是一家人）。
- **每页只过"这页真正会变的"**：知识点 ① / 套餐 ② / 强调 ③。
- **简单页**（封面/背景/收口）：① ② 近默认 → 明示给用户一眼否（"就一个核心点、走 PPT 套餐，对吗？"），不逐项审问；这页往往只有强调是真选择。
- **内容重的页**（机理图/方法设计）：① 知识点、**尤其 ② 套餐档是真决策** → 必须爆成可点选项，别默默替用户定。
- 目标：一套 N 页 ≈ 每页 0–1 个真问题 + 一个强调选择，而不是 4N 个问卷。

## 第 3 步 —— 整合成 prompt（别限太死，留 image-2 发挥空间）

按**装配顺序**（约束在前 → 主体在后 → 祈使收尾）拼 prompt，**密度锚来自第 2 步选的套餐**（科普透气、详细/journal 留白 ≤20% 且每区露内部）：

1. **注入两条硬约束**（文字一次画全 + 防指令外泄）——永远全套。
2. **蒸馏到最少图元/标签**（核心能力 = 蒸馏不是搬运）：先过白名单（删掉模型自补的近义词/背景词/泛称），再把标签压到最少（每个问"去掉它图还成立吗"），把**留下的所有标签（含中文）逐字写进 prompt、一次画全**；字别太多、Arial 型 sans、短词优先降乱码，但**不留空、不后期叠**。`简陋 ≫ 乱码`：怕糊就多抽几张挑，别预先阉割到所选套餐档以下。
3. **套 `Subject → Setting → Lighting → Text → Style` 填槽**：把内容**转成真实技术图元/结构**（token chips、矩阵/向量、attention map、score bar、pipeline panel…），别用空心方框拼。**🔴 Subject 必须来自内容地图**——隐喻/版式/质感只能**忠实承载**它，**不许为好看编出地图里没有的假元件**。
4. **从灵感库挑几样**（配色/版式/质感/signature）——按概念变化、别全套；**配色优先从主体真实材质取**（反千图一面第一闸）。
5. **补非文字完成度、别盲目加密**：标签砍少后，用子面板边界、stage header、微型矩阵/热图/散点、真实样本缩略图、图例、编号、局部放大补回专业完成度。补到"讲清楚且完整"就停（**清晰 > 完整(下限) > 密度**），别为"像 Figure 1"硬塞满版。
6. **🔴 构图交还给模型、别锁版式别逼噱头**：prompt 给 GPT「内容地图 + 护栏 + 第 2 步那句强调 note」就够，**排布/比例/具体构图交还模型**（`clean professional schematic, compose & scale freely`），别逐格规定盒子位置——实测硬塞版式比自由体还差。标准机制图常规版式（面板 / 左→右）默认就好；**求异只靠真实图元 + 内容正确，绝不靠机器 / 天平 / 隐喻噱头。skill 在这步是护栏，不是构图导演。**

**装配时一并把这些"画法红线"写进 prompt**（既是画法、也是第 4 步要查的）：
- **正式度**：用真实技术图元承载机制，别拟物成发光/玻璃/玩具机器；`lively` 卡通/发光只在明确要外宣/封面才开（但"正式"≠ 冷淡稀薄，专业图照样可多彩、精致、信息密）。
- **反俗气 + 去标签测试**：每个概念画成**它真实的对象/形态**，**盖掉文字标签还认得出**（把"打分"画成真分数条、"注意力"画成真热力小图、"分叉"画成真箭头收敛/发散）；别把概念画成"贴标签方框 + 箭头"或堆叠特征层网格，别通篇"图标 + 一个词"。
- **共享元件画成一个**：两条路命中同一个东西（同一 ID/节点/区域）→ 画**一个共享元件、多线汇入它**（prompt 写 `draw it ONCE, both arrows merge into it`），别画两个副本各指同一目标。
- **图内自洽（输入↔输出守恒）**：每个输出/结论都有配对输入；并列例子各画全 `input→output`，别留"有输出没输入"的半截。
- **连接线两端锚定**：每条线/箭头写明 `from X to Y`，不许起点/终点从空白或画面外悬空。
- **视觉详略跟 focus 走**：精致预算花在焦点元件（做大、形象化当视觉主角），配角连接线用箭头带过，别全图一个详略。

### 输出契约：一个 `figure-prompt.md`（Codex 端再 + 一张生成的图）

> **Claude 端**：产物 = 这个 `.md`。　**Codex 端**：产物 = 这个 `.md` ＋ 当场用原生文生图生成的图片，同一次回复一起给（在三道闸答完之后）。

**🔑 文件正文 = 纯 prompt，一键全选即可复制；元信息全进文件名，别混进正文。** 用户习惯 `Cmd/Ctrl+A → 复制 → 粘到出图器`，所以**正文里只能有出图 prompt 本身**——不要 `# 标题`、不要"诉求坐标"、不要"使用提示"混在里面。

- **正文**：从 prompt 第一行到最后的收尾祈使句，**仅此**，干净到可整段直接粘。
- **📛 文件名规范（严格、防冲突、信息自带）**——名字部分全下划线，点只留给扩展名：
  ```
  <concept>_<intent>_<package>_<YYYYMMDD>_<HHMM>.figure_prompt.md
  ```
  - **concept**：主题缩写、下划线连接、全小写（如 `misspelled_word`、`skill_vs_mcp`）。
  - **intent**：`understand` | `produce`。　**package**：`scicomm` | `ppt` | `detailed` | `journal`。
  - **时间戳**：`YYYYMMDD_HHMM`（当前时间）——防重名冲突的关键，每次必带。
  - 例：`misspelled_word_understand_scicomm_20260625_1430.figure_prompt.md`
  - 规矩：**全小写 + 全下划线 + 零横杠**；点只出现在扩展名 `.figure_prompt.md`。
- **需要给用户的提示/坐标**（套餐、强调、出 3–4 张挑一张、保留 fucntion 拼错等）**写在聊天回复里，不写进文件**。
- **📍 默认最后一行给绝对路径**：回复最后一行展示该 prompt 文件（Codex 还有生成图片）的绝对路径，方便用户定位。

## 第 4 步 —— safeguard（出图前过一遍红线，不过就重写）

**流程闸**
- **调用透明了吗**？第一句说了用 storytelling-figures 吗？
- **三道选择都走了吗**？① 知识点 ≤5（没替用户默认 focus）② 套餐 ③ 强调点——**漏知识点那步、或闷头直接出图 = 失败**（除非用户明说"你定/直接画"）。
- **意图判对了吗**？理解型有没有被一堆选项淹没；产出型有没有重讲他早懂的内容。
- **产物对平台吗**？Claude 端就一个 prompt.md（没手搓 SVG/PNG）；Codex 端把图也生成了、和 prompt 一起交（停在 prompt = 没做完）。

**内容闸**
- **每个元件对应内容地图一条真实内容吗**？有没有为好看编的假元件？（content-first）
- **可见词过白名单了吗**？有没有把纯背景/泛称（路过的工具名、"图""方法"）误当概念节点、或引入用户没要求的词？
- **图内自洽吗**？每个输出都有配对输入？并列例子各画全 `input→output`（别留"有 `logit(模型)` 没'模形'输入"的半截）？
- **（深档）是真实机制名还是泛动词**？图上是 attention / 余弦相似度 / top-k / 动态注入，还是停在"识别/匹配/处理"？停在动词 = 没钻够，回去补。
- **术语就地解释了吗**？logit / embedding / softmax 这类标签配了一句大白话注释（"logit = 候选词分数，越高越可能"）吗？

**画面闸**
- **标签全画进图了吗**（含中文、不留空、字不过多）？
- **数据流闭合 + 无悬空线**？每条线/箭头两端都锚到具名元件？有没有凭空冒出的元件、起点悬空的线？
- **正式度够 / 没越权导演吗**？有没有拟物成发光/玻璃/玩具机器、**天平/跷跷板这类隐喻噱头**？**prompt 有没有越权锁死版式（该交给模型）、或为求异硬逼一个噱头框架**？
- **够美 / 反俗气吗**？有没有退化成"方框 + 箭头 / 堆叠特征层"那种过时 AI 示意图脸？每个概念画成真实对象（盖标签还认得出）了吗？
- **完成度对得上套餐 + 密度合适吗**？详细/journal：留白 ≤20%、每区露内部结构？反过来有没有为密而密？**清晰是主、完成度只是下限。**
- **配色协调吗**？一套调性、无突兀撞色（冷色图别塞深绿）？暖色 accent 只给焦点。
- **求异了吗**？这张和上一张不一样吗？唯一信息够清楚吗？能更少元件吗？
