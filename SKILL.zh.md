# storytelling-figures

把**一段对话里聊出来（或正卡着想搞懂）的概念** → 一个 `figure-prompt.md`（纯文本 prompt），粘给 GPT image-2 / nano-banana 出**叙事/概念/科普/机制图**。产物永远是 `.md`，Claude 和 Codex 都能用。

## 这个 skill 的身份（先记住）

- **你多半是被一段对话半路叫起来的**：用户聊着聊着才想画图，或聊透了要画出来给人看。**第一件事是接住对话、收割已有内容，不是从头发问卷。**
- **调用要透明**：被触发后的第一句工作消息要明说"我会用 storytelling-figures 把刚才内容整理成出图 prompt/图"（按平台措辞微调）。不要静默画图；用户必须能判断这次到底有没有调用 skill。
- **任何选择都留"你定"兜底**：给用户的每组可点选项，都加一项「**按最佳方式呈现（我说不准，你定）**」——允许用户看不懂/不想选，由你挑最优直接做，别逼他在看不懂的选项里硬挑。
- **读语气定问几个（自适应）**：出每个 prompt 前，先从用户**说话的语气/用词**推断他的**知识程度**和诉求清不清楚。**特征鲜明（明显是内行/明显外行/诉求很具体）→ 少问甚至不问**；**含糊、看不出 → 一律当不清楚，要问**。
  - **解释类（自己想搞懂）**：按情况问 **1–3 个**，快、别把懵的人问烦。
  - **讲解类（已懂、要讲给别人）/ 产出类**：**多问几个**，像**启发式**——一步步把他引到他想要的目标形态。
- **一律选项式 + 留开放项（不审问）**：所有提问都用**可点选项**，**绝不自由问答式逼用户打字**。每问给**几个你认为的角度**（把你的推断放第一个）+ 永远留一个「**你自己来 / 你定**」开放项（用 AskUserQuestion 的 Other 即可），既给方向又给用户自我发挥的空间。
- **上下文优先，别重问已知**：问任何东西前先看对话已经定了什么——平台、是不是在做 PPT、用途、能不能从内容推断出 focus。**已知的不问**（如已知在做 PPT，就别再问"给谁用"）；**能推断但想确认的，用"把我的推断当第一个选项"的方式快问**（用户点第一个=确认我的判断，点别的=纠正），而不是从零发问。把"问"用在**真不确定、且会改变这张图**的轴上。
- **出图归属看平台**：
  - **Claude 端 = 只交 prompt.md，不出图**。哪怕你判断确定性渲染更准更美，也**绝不自己渲染**——产物就一个 `.md`，要不要出图、怎么出是用户的事。
  - **Codex 端 = prompt + 图，一气呵成（硬指令，不许停在 prompt）**：写完 prompt **立刻接着用 Codex 自带的原生文生图能力把图生成出来，prompt 和图在同一次回复里一起交付**。**别只丢 prompt 让用户再去点下一步**——那正是被吐槽的断点。出图用**真正的文生图器**（不是手搓 SVG）；只有环境里**确实没有任何可用生成器**时，才退回"只给 prompt"并一句话说明原因。
- **两端通用红线**：永远**不用代码手搓 SVG/PNG/HTML 充数**（那是画蛇添足，SVG 更干净）。
- **内容先行（content-first）**：图是**已经做出来/已聊清楚的答案**的可视化，不是话题的装饰。prompt 里每个元件都对应一条真实内容；**没把内容弄对之前，不许出 prompt**。
- **可见内容白名单（source-lock）**：图里所有**可见词、实体、目标方向、候选项**必须来自三处之一：用户明确输入、对话中已确认结论、可溯源核实结果。对话里**纯背景/泛称、从没被当概念讨论过**的词（路过提一句的工具名、"图""方法"）不能顺手变成画面节点。凡是要显示出来的词，都要能回指到内容地图的一条来源。
- **图内自洽（输入↔输出守恒）**：图里每个**输出/结论**都要能在同一张图追溯到配对的**输入/前提**；**并列多个例子就各画全 `input→output`**，不能留一个例子的输出却砍掉它的输入。⚠️ 别和上面的"来源污染"搞混：词**没来源**（纯背景）→ 删；词是**真实并列例子、只缺输入** → **补输入**别删输出。典型翻车：输出冒出"模型方向 / logit(模型)"、输入却只有 `fucntion` 碎片没有"模形"——这不是污染，是图**漏了输入、不自洽**；修法是补"模形"那条输入，或只画 `function` 一个例子。
- **连接线/箭头两端都要锚定（悬空线绝对踩雷）**：图里**每一条线/箭头/虚线，两端都必须连到一个具名元件**——既有明确的**起点**也有明确的**终点**，不许从空白处、画面外、或一团模糊里飘出来。典型翻车：几条虚线扑向某个点，但**另一端没有起点**，读者根本不知道"什么流进来了"——这跟"凭空冒出来的元件"同罪，讲不清就废。prompt 里要写明每条连接线 from X to Y。
- **正式度（默认开，一条原则代替清单）**：科研/组会/方法图工具，默认正经、克制、技术感。原则就一句——用**真实技术图元**（token chips、矩阵/向量、attention 小图、logit 条、真实样本缩略图…）承载机制，**别把抽象过程拟物成发光/玻璃/玩具机器**。`lively` 只在用户明确要外宣/封面时才开。
- **反"俗气模板"（失败模式，硬规则）**：**把概念画成"贴标签的方框/方块 + 箭头"、或堆叠的"特征层"方块网格——一律不过关**，那是过时的 AI 示意图脸（"现在都什么年份了"）。每个概念要画成**它真实的对象/形态**（真切片、真分子、真曲线、真矩阵、真界面…，draw the actual thing），配现代构图与真实质感/层次。分区面板和连接箭头只能当"连接组织"，**不能让一个 labeled box 本身充当概念**。
- **「去标签测试」（判俗气的硬标尺）**：把每个元件的**文字标签盖掉**——它**还看得出是什么吗**？看不出（只剩通用图标 / 空盒，全靠文字代劳）= 装饰，**不过关**。每个元件要**画出它的真实内容/结构**，盖了标签也读得出（典型正例：把"方向选择"画成几行真 ASCII 草图、把"打分"画成真分数条、把"分叉"画成真箭头收敛/发散）。两条延伸：① **别通篇"图标+一个词"**——单调且空，图标+文字描述可以用但不能从头用到尾；② **流程/分叉用干净箭头，别拟物成铁轨/齿轮/机器**（哪怕 PPT 也不许这么不正经）。
- **视觉详略跟着 focus 走（该花哨花哨、该沉住沉住）**：把"画得精致、形象化"的预算**花在核心/焦点元件**上——它们要有**真正图案化、形象化的示意**（把概念画成一眼看懂的视觉，如 converge=多点汇聚成一、diverge=一点发散成多、一条逐步收窄的选择链），**做大、当视觉主角**。**非核心的连接线/配角要沉住**：一根连接线**用箭头就够了**，别当重点去装饰；catch/guardrails/output 这类配角简洁带过。**别全图一个详略**——那既浪费在不重要处，又埋没了核心。这就是 focus（详略分布）的**视觉版**：focus 选中的不只是"标签多写"，而是"**画得更形象、更精致**"。
- **清晰是主、完成度只是下限**：讲清楚永远第一。完成度只保证**不半成品**（别只剩几个空框+箭头+大片空白、没有内部结构），**不是追密度**——**禁止为"像 Figure 1"强行加密**。中等密度能讲明白就停在那，功夫花在自洽和用途完整上，不是塞满子面板。
- **🎯 完整度跟用户选的 coverage 走，别因怕糊而降档（实测校准）**：**完整还是简洁是用户选的**（coverage 轴）——他要简洁干净就**简洁**，他要 detail 你就**真给 detail**。**别自作主张定一个"默认完整"或"默认简洁"。** 真正的失败模式 = **用户选了"全面/完整"，你却因为怕糊交了张简陋图（降到他选的档以下）**。实测：image-2 对**清晰文字**承载力比想象强（10+ 标签 + 子面板都渲得清清楚楚），所以"砍标签"是砍**噪声**、**怕糊不是把图降到所选档以下的理由**——按他选的密度给，**糊就多抽几张挑**，别预先阉割。`简陋 ≫ 乱码`：前者几乎必中、后者多抽即解。
- **晋升测试（防过拟合 / 反千图一面）**：想把某张图的教训写成规则前先问——它是**失败模式**（乱码 / 不自洽 / 用词错 / 逻辑断 / 指令外泄）还是**风格偏好**（该不该某种隐喻、该多密、像不像某篇 Figure 1）？**失败模式才可升成规则；风格偏好只进灵感库菜单。** 本 skill 只挡奇怪情况，其余一律放给 image-2 和用户发挥。
- **核心能力 = 蒸馏，不是搬运**：把一长段对话压成 **ONE message + 最少必要图元/标签**。image-2 标签越多、小字越多越糊——**宁可砍枝节，也别把整段对话塞进一张图**。一张图只扛一个核心点；每个标签都问一句"去掉它图还成立吗"，能去就去。
- **一次成图、不后处理**：所有标签（含中文）逐字写进 prompt，让图一次画全。**接受适当乱码**，但用最强的"照写清楚"指令压低——**绝不**留空白让用户后期补。
- **发挥 image-2 出"专业美图"**：默认追求 publication-quality schematic / polished Keynote figure，不是儿童科普插画。可以有精致质感、轻投影、清晰层次；别把抽象机制拟物成发光玩具机器（见上"正式度"那条原则）。**方框+箭头那种简陋示意图是 SVG 的活；过度插画化、比内容更抢眼，也同样是写砸了。**
- **跨平台（Claude / Codex 通用）**：本文一律说"动作"、不绑某家工具——"用**可点选项**问"（Claude 有 `AskUserQuestion` 就用，Codex/其它没有就**纯文本列选项**让用户选）、"**派研究子 agent / 查官方源**核实"（Claude 可用 `claude-code-guide`，Codex 用自带研究/检索）。产物（prompt.md）本就跨平台。

> 边界：**按平台交付**（Claude 端只交 prompt.md；Codex 端 prompt + 原生文生图），但永远**不手搓 SVG/PNG/HTML 充数**；不做数据图表（→ `nature-figure`）；不做整套 PPT（→ `academic-pptx`）；不保"精准换单个元件"。

## ⚠️ 最高律：求异 + 求美（覆盖本文件一切其它规则）

**成败标准 = 每张图都不一样、各讲各的故事，且发挥 image-2 画得好看。千图一面 / 简陋方框 = 彻底失败。** 约束只有两层：

| 层 | 内容 | 强制度 |
|---|---|---|
| **硬约束（永远开）** | 仅两件：① **文字一次画全**（含中文、不留空、接受乱码）② 防指令外泄。这是**正确性**，与审美无关。 | 必须 |
| **灵感库（按图取用）** | 配色 / 版式 / 质感 / 隐喻……全是**菜单不是清单**。现场挑几样、**变着用**，每张图给独有 signature。 | 可选 + 鼓励变化 |

**守则**：① 别套同一份配方，宁可冒险也不要千图一面；② **别把"必扁平/无色/无 3D/细线"这类学术绘图刻板印象塞给 image-2**——专业图也要好看，但好看来自结构、真实图元、配色和层次，不来自玩具化隐喻；③ **默认偏正式（这是科研工具）**：专业 family（`ppt`/`journal`）是默认调性，`lively` 的卡通/发光/吉祥物**只在明确要科普外宣/封面时才用**——但"正式"**≠冷淡/稀薄**，专业图照样可多彩、精致、信息密；④ **密度服从清晰度**：完成度是下限、不是每题都要满版。风格名只是**调性起点**，不是模具。

## 工作流：收割 → 选知识点 → 选套餐 → 选布局 → 图工程

### Step 1 —— 接住 + 收割（被叫起来第一件事）

你大概率在一段已有对话之后才被叫起来。**别从头发问卷。** 先做两件事：

1. **收割对话**：把已聊出的内容/机制整理成「内容地图」，同时建「**可见内容白名单**」（`允许画/写的词与实体` / `禁止混入的对话泛称` / `不确定需问·需核实的词`）。例：用户只要求解释 `fucntion → function`，白名单就只有 `fucntion`、`function`、相关机制词；对话里泛指 AI 的"模型"不准变成"模型方向"或 `logit(模型)`。
2. **轻辨意图（定收敛/发散，别一上来抛一堆问卷）**：
   - **理解型**（"太难讲不下去/帮我画出来理一理"、"学懂了要讲给组会"）→ 后面布局**收敛**给 1–2 个方向、问得少。
   - **产出型**（"做 PPT 配图"、"画 journal 方法图"、"给排版灵感"）→ 布局**发散**给 2–3 个方向。
   - 拿不准就一句话问意图。

### Step 2 —— 脑暴知识点 → 选 ≤5（🛑 硬闸①，最常被漏的一步）

被调用后**立即展开脑暴**：把内容里**所有可能的知识点/子主题**摊出来，用**可点选项（控制在 5 个以内）**问用户"这张图你想突出哪几块？"。用户勾选 → 锁定"图要装什么"。
- **绝不许跳过、绝不许替用户默认 focus。** 漏掉这步、直接奔形式 = 最常见的翻车。
- 选项**从对话内容现场枚举**（不写死）；可多选，但提示**一张图最好 ≤3–4 个知识点**，多了糊。
- 每组留「**全都要 / 你定**」兜底。平台兜底：有 `AskUserQuestion` 用卡片，没有就纯文本编号列（`1. … 2. … 6. 你定`）。

### Step 3 —— 选套餐（🛑 硬闸②：四大套餐，一步定"画多满 + 长什么样"）

把原来散开的 depth / style / coverage 收成**一个"套餐"选择**——用户选**目的地**，各轴的合理默认就捆好了。留「其他/自行描述」逃生口。

| 套餐 | 给谁/干嘛 | 留白 | 最少元件 | 内部细节 | 一句目标 |
|---|---|---|---|---|---|
| **科普** | 外行 / 海报 | 45–60%（透气） | 1 主对象 + 2–4 标签 + 1 主线 + 1 句 caption | 主对象画得形象，其余从简 | 3 秒懂一个概念 |
| **PPT** | 报告幻灯 | 25–40% | 3–6 个主部件，各带小标题 + 1–2 行说明 + 清晰主流向 | 每部件画成真实对象、有内部结构，不堆顶 | 边讲边指，听众跟得上 |
| **详细描述** | 教学/自学的密集讲解图 | **≤20%，空白=没画完** | **≥6–10 个有内容区块**，每区有子结构/子步骤 | **每个概念露出内部机制**（如 attention 画到 QKV/softmax 条/残差加法），禁空盒 | 不靠口头也自洽读懂 |
| **journal-figure 启发** | 投稿论文 figure | ≤20%，克制 | 分面板 a/b/c + 图例 + caption，出版规范齐全 | 每面板用真实图元、露机制，克制配色(如 Okabe-Ito) | 审稿人级，**且多给 2–3 个布局启发让你挑** |

**三条硬规则（专治"详细不够详细"）**：
1. **详细 / journal 档：留白 ≤20%，把留白当缺陷信号**——看到空白就继续填。
2. **详细 / journal 档：每个区域必须露内部结构，禁止空盒子/纯标签方块。**
3. **每档都按"最少元件数"下限给**（可数 floor），堵偷懒。
> 但 **清晰 > 完整(floor) > 密度**：详细档再满也得**严格易读**，不许糊；不为"像 Figure 1"硬加密。
> 一句话区分 **详细 vs journal**：详细=密度顶满的"讲解图"（画风自由、可上色可隐喻）；journal=出版规范的"论文图"（克制、分面板、额外给构图灵感）。

**journal 套餐的「三保」**（正式场景最易翻车）：① **保求异**——靠"版式按方法结构选 + 真实图元 + 语义配色(一模态一色) + signature=真实焦点"差异化，并在 prompt 明令避开 image-2 俗相（渐变方框流/发光节点/假 3D 铬光）；② **保正确**——元件来自内容地图或用户原话，可疑处核实，**绝不为构图编出方法里没有的步骤/箭头**；③ **保不胡写**——标签逐字来自真实术语、绝不臆造，拿不准宁可问/留白给用户（正式投稿最终文字保真靠矢量工具，image-2 只交布局灵感）。

### Step 4 —— 选布局方向（嵌套关系 + note，少 ASCII）

给 **1–3 个真正不同的布局方向**让用户挑（理解型 1–2、产出型 2–3），再深做选中的那个。这步专治"出来不符合预期 / 一股俗气味"。**第一张图前必须真的把方向弹给用户**（可点选项；除非用户明说"你定/直接画"）。

每个方向 = 三段，**重点在后两段，ASCII 别堆**：
1. **位置骨架（轻 ASCII，≤4 行）**：只示意"谁在上/左、流向往哪"，并注明**这只是占位、不是画风预览**。形状贴短词标签、列对齐、箭头标流向，别留裸 `●`/`□`。
2. **嵌套关系（大→小）**：讲清几块信息**谁包含谁、读图顺序**，要有道理。例：
   ```
   时间轴
    └ 两条泳道(成像 / 测序)
       └ 每条 5 个里程碑节点
          └ 每节点 = {方法名 · 年份 · 一句贡献}
   ```
3. **note（一句）**：「**更体现 X，弱于 Y**」——点出这版强调什么、舍什么，让用户**按"想强调什么"选**。**note 仅供选方向，不写进图。**

- 方向之间要**本质不同**（不同母题/构图/强调点），不是同一张图的微调。
- 🚫 每个方向都**不准是"方框+箭头/堆叠特征层/方块网格"**——把概念画成**真实对象**（见身份段「反俗气模板」）。
- 用**可点选项**给（有 `AskUserQuestion` 就用、`preview` 里放位置骨架并排比较；没有就纯文本编号列）。每组留「**按最佳方式呈现（你定）**」兜底；用户说"随便/你定/直接画"就自己选最贴的一个直接做，附一句"想换方向再说"。

### Deck 模式（旁路：一套图 PPT / 海报多页 / 系列图，不是单张）

当任务是**一套图**而非单张，别把单图流程 ×N（那会变成几十个问题）。改成"**默认设一次 + 每页只问会变的**"：
- **deck 级默认，开场设一次、之后不重问**：意图（如在做 PPT = 产出型）、style 家族 + 密度、house style（统一视觉线，保证 N 页是一家人）。这些**已知就别再逐页问**。
- **每页只过"这页真正会变的"**：知识点（Step 2）/ 套餐（Step 3）/ 布局方向（Step 4）。
- **简单页**（封面 / 背景 / 收口）：知识点、套餐近默认 → **明示出来给用户一眼否**（"就一个核心点、走 PPT 套餐，对吗？"），不要逐项审问；这页往往**只有布局方向是真选择**。
- **内容重的页**（机理图 / 方法设计 / 复杂案例）：知识点、**尤其套餐档是真决策** → 必须爆成可点选项，别默默替用户定。
- 每页都**记住 deck 默认**，只把增量问题抛给用户。目标：一套 N 页 ≈ 每页 0–1 个真问题 + 一个布局选择，而不是 4N 个问卷。

> **🎚️ 每页两道硬闸照走（内容重的图，强制）**：讲**机制 / 方法 / 实现**的页，**知识点（Step 2）和套餐（Step 3）必须作为可点选项弹给用户**（把你推断的当第一个）。**不许默默替用户定**——选哪些知识点、选哪个套餐档，直接决定这页画什么、画多深多满，是用户最该掌控的旋钮。

### Step 5 —— 图工程：→ figure-prompt.md

按**装配顺序**（约束在前→主体在后→祈使收尾）拼 prompt（**密度锚来自 Step 3 选的套餐**：科普透气、详细/journal 留白≤20% 且每区露内部）：
1. 注入**硬约束**（文字一次画全 + 防指令外泄）——永远全套。
2. 从**灵感库挑几样**（配色 / 版式 / 质感 / signature）——按概念变化、别全套；**配色优先从主体真实材质取**（反千图一面第一闸）。
3. 套 `Subject → Setting → Lighting → Text → Style` 填槽，把内容**转成可渲染的技术图元/结构**，别用空心方框拼；专业默认先想真实图元（token、矩阵、向量、attention map、score bar、pipeline panel），不是先想隐喻机器。
   - **🔴 红线：Subject 必须来自内容地图**——每个画进去的元件都对应一条真实内容；隐喻/版式/质感只能用来**忠实承载**它，**不许为了好看编出地图里没有的假元件**。专业场景里，隐喻最多是轻微结构化表达，不能变成玻璃机器/发光盆地这类主角。
4. **先过白名单，再砍标签预算**：所有可见标签/实体/候选项/语义方向必须来自白名单；任何模型自己补的近义词、背景词、泛称都删掉，或改成不可见图元。然后把标签数压到最少（每个都过"去掉它图还成立吗"），再把**留下的所有标签（含中文）逐字写进** prompt、一次画全；**字别太多、字号用常见档、Arial 型 sans、短词优先**降乱码，但**不留空、不后期叠**。"一次画全"约束的是**留下来的**那些，不是把对话全塞进去。
5. **再补画面完成度，别盲目加密**：标签砍少之后，用**非文字细节**补回专业完成度：子面板边界、stage header、微型矩阵/热图/散点/条形图、真实样本缩略图、图例、编号、局部放大、虚线分组。补到"讲清楚且完整"就停；不要用更多文字填密度，也不要把单一路径题硬塞成满版 Figure 1（**清晰是主、完成度只是下限**）。版式 archetype 永远**跟着这个概念自己的结构走**，别套固定模板。

### 冷启动兜底（少数：上来就"帮我画个 X"、没有前置对话可收割）

没有对话可收割时，才走完整老流程：
1. **自己做答案（Phase A）**：把概念真机制过准过全，产内容地图。
2. **走 Step 2→3→4**：用**可点选项**（有 `AskUserQuestion` 就用，没有就纯文本列）问 **知识点（≤5）→ 套餐 → 布局方向**，措辞全用**大白话**（聪明但外行的读者也得看懂；术语只在你心里）。
3. 进 Step 5 图工程。
> 枚举知识点、选套餐档、推版式 archetype 的方法见上文 Step 1 / Step 2 / Step 3。

## §深档硬闸（研究/最深档，或理解型要钻到实现层时）

Phase A 必须**钻到实现层**——具体的数据结构、算法、判定分支，每条**可溯源**。**严禁用"意图识别 / 检索匹配 / 相似度 / 处理"这类含糊动词糊过去**（真翻过车：把"默认 = 注意力直选、无检索"误写成"相似度打分"）。**不许"我恰好知道"**——深档就得真去查官方源 / 派一个**研究子 agent** 核实（Claude 可用 `claude-code-guide`，Codex 用自带研究/网页检索），把真机制名（attention、余弦相似度、JSON 工具清单、top-k、动态注入…）落到图上（呼应 CLAUDE.md：数值/机制必须可溯源）。

## 输出契约：一个 `figure-prompt.md`（Codex 端再 + 一张生成的图）

> **Claude 端**：产物 = 这个 `.md`。　**Codex 端**：产物 = 这个 `.md` **＋ 当场用原生文生图生成的图片**，同一次回复一起给。

**🔑 文件正文 = 纯 prompt，一键全选即可复制；元信息全进文件名，别混进正文。**
用户习惯 `Cmd/Ctrl+A → 复制 → 粘到出图器`，所以**正文里只能有出图 prompt 本身**——不要 `# 标题`、不要"诉求坐标"、不要"使用提示"、不要任何说明文字混在里面（那些一全选就被一起复制，污染 prompt）。

- **正文**：从 prompt 第一行到最后的收尾祈使句，**仅此**，干净到可整段直接粘。
- **📛 文件名规范（严格、防冲突、信息自带）**——**名字部分全下划线，点只留给扩展名**：
  ```
  <concept>_<intent>_<package>_<YYYYMMDD>_<HHMM>.figure_prompt.md
  ```
  - **concept**：主题缩写、**下划线连接**、全小写（如 `misspelled_word`、`skill_vs_mcp`、`qtl`）。**不要横杠。**
  - **intent**：`understand` | `produce`。　**package**（Step 3 选的套餐）：`scicomm` | `ppt` | `detailed` | `journal`（没问就用推断档）。
  - **时间戳**：`YYYYMMDD_HHMM`（取当前时间）——**防重名冲突的关键**，每次必带。
  - 例：`misspelled_word_understand_scicomm_20260625_1430.figure_prompt.md`
  - 规矩：**全小写 + 全下划线 + 零横杠**；点**只**出现在扩展名 `.figure_prompt.md`。一个文件名读得出"画了啥、给谁、什么套餐、何时出"，且多次出图不互相覆盖。
- **需要给用户的提示/坐标**（知识点/套餐/出 3–4 张挑一张、保留 fucntion 拼错等）**写在聊天回复里**，**不写进文件**。
- **📍 默认最后一行给绝对路径**：回复的**最后一行**展示**该 prompt 文件（Codex 还有生成图片）的绝对路径**，方便用户直接定位/复制——除非用户另有要求。

```
<工程化后的英文 prompt：约束块（SOURCE-LOCK + TEXT + DO NOT RENDER META）→ Subject…Style → 收尾祈使句>
（整个文件就这一块，没有任何别的行）
```

## 交付前自检

① **调用透明了吗？**第一句工作消息有没有明说用了 storytelling-figures？
② **意图判对了吗**——理解型有没有反而抛一堆版式选项淹没本来就懵的人？产出型有没有去重讲他早懂的内容？
③ **每个元件都对应内容地图里一条真实内容吗？**有没有为好看编的假元件？（content-first 红线）
④ **可见内容过白名单了吗？**图里有没有把**纯背景/泛称**词（路过提的工具名、"图""方法"）误当成概念节点？有没有引入用户没要求的词？
④b **图内自洽（输入↔输出守恒）**：每个输出/结论在图里都有配对的输入吗？并列例子各画全 `input→output` 了吗？——别留"有输出没输入"的半截例子（典型翻车：画了 `logit(模型)` 却没画"模形"输入）。修法：补输入，或删这条例子。
⑤ **（深档专属）图里是真实机制名（attention / 余弦相似度 / top-k / 动态注入）还是停在泛泛动词（识别/匹配/处理）？** 停在动词 = 没钻够，回去补。
⑥ 科学准确　⑦ 清晰可读　⑧ **标签全画进图了吗（含中文、不留空、字不过多）**　⑨ 构图
⑩ **数据流闭合**：每个元件的输入从哪来、输出到哪去都有箭头吗？有没有**凭空冒出来的元件**？**每条线/箭头/虚线是不是两端都锚定到具名元件**——有没有**起点悬空的线**（典型踩雷：虚线扑向某点但另一端不知从哪来，讲不清）？
⑪ **正式度够吗？**有没有把抽象机制拟物成发光/玻璃/玩具机器、或过度卡通/戏剧化？默认应像正经组会/PPT/方法图。
⑫ **配色协调**：全图一套调性吗？有没有**突兀撞色**（如冷色图里塞一块深绿）？暖色 accent 只留给焦点。
⑬ **术语就地解释**：图里的专业术语标签（logit、embedding、softmax…）有没有配一句大白话注释（"logit = 候选词分数，越高越可能"）？光甩术语 = 没讲清。
⑭ **够不够美 / 反俗气**：是不是又退化成"方框·方块 + 箭头 / 堆叠特征层"那种过时 AI 示意图脸？每个概念有没有画成**真实对象**（而不是 labeled box）？盒子流 / 俗气 → **不过关，重写**。
⑭c **三道选择关都走了吗？（硬闸）**第一张图前**真的用可点选项**走过：① **知识点 ≤5**（让用户选突出哪几块，没替他默认 focus）② **套餐**（科普/PPT/详细/journal，定画多满）③ **布局方向 1–3 个**（每个含位置骨架 + 嵌套关系 + 一句 note）？**漏掉知识点那步、或闷头直接出图 = 失败**（除非用户明说"你定/直接画"）。
⑭b **完成度对得上所选套餐吗、密度合适吗？**有没有简陋空框 + 大片空白、没有内部结构？**详细/journal 档：留白是否 ≤20%、每区是否露内部结构**（别交了张比所选套餐更简陋的图）？反过来有没有为密而密、妨碍一眼看懂？**清晰是主、完成度只是下限。**
⑮ **产物对不对平台**：Claude 端就一个 prompt.md（没手搓 SVG/PNG）；**Codex 端有没有把图也生成出来、和 prompt 一起交（停在 prompt = 没做完）**。
外加 **唯一信息？能更少元件吗？**
